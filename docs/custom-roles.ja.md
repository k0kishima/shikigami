# カスタムロール

<p align='center'>
  <a href='./custom-roles.md'>English</a> | 日本語
</p>

## 概要

Shikigami にはビルトインロール（Coder, Reviewer, Tester 等）が用意されていますが、プロジェクトによってはこれらだけでは不十分な場合があります。  
カスタムロール機能を使うことで、プロジェクト固有のロールを定義し、タスクフォースに組み込むことができます。

## ユースケース

- フロントエンドとバックエンドで Coder を専門化したい
- データエンジニアやインフラエンジニアなど、ビルトインにないロールが必要
- プロジェクト固有のドメイン知識を持つ専門ロールを作りたい

## 配置場所

作業ディレクトリの `.shikigami/roles/` にロールテンプレートを配置します。

```
<作業ディレクトリ>/
  .shikigami/
    roles/
      frontend_coder.md
      backend_coder.md
      data_engineer.md
    contexts/          ← ロール別コンテキスト（併用可能）
      frontend_coder.md
      backend_coder.md
```

## テンプレート書式

カスタムロールは[ロール定義ガイド](./role-definition-guide.ja.md)に従って記述してください。  
ビルトインロールと同じ書式（Description, Goal, Backstory, Constraints, Tools, Temperature）を使用します。

## ビルトインロールのオーバーライド

カスタムロールがビルトインロールと同名の場合、カスタムロールが優先されます。

例えば `.shikigami/roles/coder.md` を配置すると、ビルトインの `roles/coder.md` の代わりにカスタム版が使用されます。  
これにより、ビルトインロールの振る舞いをプロジェクトに合わせて調整できます。

## コンテキストファイルとの併用

カスタムロールでも `.shikigami/contexts/` によるコンテキスト管理が利用できます。  
ロールテンプレートにはロールの定義（ペルソナ、目標、制約）を、コンテキストファイルにはプロジェクト固有の情報（規約、方針）を記載するという使い分けを推奨します。

| ファイル | 記載内容 |
|----------|----------|
| `.shikigami/roles/frontend_coder.md` | ロールの定義（React 専門のフロントエンド開発者としてのペルソナ等） |
| `.shikigami/contexts/frontend_coder.md` | プロジェクト固有の情報（コンポーネント設計規約、CSS 命名規則等） |

## オーケストレーターの動作

オーケストレーターはタスクフォースを提案する際、ビルトインロールとカスタムロールの両方を候補として使用します。

1. ビルトインロール（`$ROLES_DIR/`）を一覧する
2. カスタムロール（`.shikigami/roles/`）を一覧する
3. 同名のロールはカスタムで上書きする
4. 統合されたロール一覧をもとにタスクフォースを設計する
